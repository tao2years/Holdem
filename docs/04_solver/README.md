# Solver 设计与实现

本目录用于记录 Solver 的设计思路、核心算法与工程化组织方式，强调“从简到繁”的可迭代演进。

## 核心模块（分）

- `game/`：游戏规则、动作与状态定义
- `tree/`：状态树与行动树构建
- `core/`：核心算法（CFR / CFR+ / MCCFR）
- `training/`：训练与迭代流程
- `eval/`：评估与对比工具

## 当前实现（分）

- 提供通用 CFR 框架
- 提供 Kuhn Poker 作为最小可运行示例

## 小结（总）

Solver 将以最小可运行版本为起点，逐步扩展到德州扑克场景的抽象与性能优化。
