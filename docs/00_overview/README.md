# 总览与路线图

本目录用于提供仓库整体结构、学习路径与建设路线的统一入口，帮助快速理解"知识体系 + Solver 实现"的双线目标。

## 本仓库要解决什么问题（总）

- **建立可持续迭代的德州扑克 GTO 知识库**：系统整理从基础到进阶的 GTO 理论，形成可检索、可验证的知识体系
- **以最小可运行 Solver 为核心，逐步扩展到可用工具**：从 Kuhn Poker 开始，逐步扩展到 Leduc、德州抽象，最终支持实战分析
- **支持学习、复盘与工程实践的闭环**：理论文档、代码实现、实验验证三者相互印证，形成完整的学习与实践路径

## 仓库结构（分）

### 文档体系（docs/）

- **00_overview/**：总览、路线图、GTO 训练手册入口
- **01_basics/**：术语、规则、赔率、EV、底池计算等基础工具
- **02_gto_theory/**：GTO 核心概念、范围频率、42 原则、训练手册
- **03_advanced/**：翻牌前范围、翻后策略、下注尺度、读牌与底池动态、案例研究
- **04_solver/**：Solver 架构、CFR 算法、Kuhn 示例
- **05_experiments/**：实验模板与结果记录
- **99_notes/**：碎片笔记与临时想法

### 代码实现（solver/）

- **core/**：CFR 等核心算法
- **game/**：博弈定义（Kuhn、Leduc 等）
- **tree/**：状态树与行动树构建
- **training/**：训练流程与配置
- **eval/**：评估与对比工具
- **cli/**：命令行入口

### 数据目录（data/）

- 训练结果、策略文件、实验数据

## 学习路径（分）

### 新手路径

1. **基础知识**：阅读 `docs/01_basics/` 建立术语与计算基础
2. **GTO 入门**：阅读 `docs/02_gto_theory/core_concepts.md` 理解均衡概念
3. **实战训练**：跟随 `docs/00_overview/gto_handbook.md` 进行系统训练
4. **代码验证**：运行 `solver/cli/run_kuhn.py` 理解 CFR 训练流程

### 进阶路径

1. **策略深化**：深入学习 `docs/03_advanced/` 各章节
2. **Solver 实现**：阅读 `docs/04_solver/` 理解算法与架构
3. **实验验证**：使用 `docs/05_experiments/` 模板记录与对比实验
4. **扩展开发**：基于现有框架扩展新博弈或优化算法
