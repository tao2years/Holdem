# 实验与验证

本目录记录实验流程、参数、结果与对比结论，保证 Solver 结果可复现、可解释。

## 本节目的（总）

- 统一实验记录格式，确保可复现
- 建立算法对比与性能评估标准
- 形成可持续的迭代验证流程

## 实验分类（分）

### 算法验证实验

- **Kuhn Poker 收敛性验证**：验证 CFR 算法在简单博弈中的正确性
- **Leduc Poker 策略对比**：对比不同算法（CFR、CFR+、MCCFR）的效果
- **理论解对照**：与已知最优策略对比，验证实现正确性

### 性能评估实验

- **训练速度对比**：不同算法的训练时间与收敛速度
- **内存占用分析**：大规模博弈的内存需求
- **并行化效果**：多线程/多进程训练的加速比

### 策略分析实验

- **策略稳定性测试**：多次训练的策略一致性
- **频率分布分析**：各信息集合的策略频率分布
- **期望收益验证**：零和博弈的收益平衡性

## 实验模板（分）

使用 `experiment_template.md` 记录每个实验，包含：

- **实验配置**：目标、模型、参数、数据
- **过程记录**：训练轮次、收敛表现、关键观察
- **结果与结论**：指标、结论、下一步

## 实验流程（分）

### 1. 实验设计

- 明确实验目标与假设
- 确定对比基准与评估指标
- 设计实验配置（参数、数据、环境）

### 2. 实验执行

- 记录实验环境（Python 版本、硬件配置）
- 执行训练并记录过程
- 保存策略文件与日志

### 3. 结果分析

- 计算评估指标
- 可视化策略与收敛曲线
- 对比理论解或基准结果

### 4. 结论与下一步

- 总结实验结论
- 指出发现的问题
- 规划下一步实验

## 数据管理（分）

### 实验数据存储

- **策略文件**：`data/strategies/experiment_name/`
- **日志文件**：`data/logs/experiment_name/`
- **结果报告**：`data/reports/experiment_name/`

### 版本控制

- 使用 Git 标签标记重要实验版本
- 记录实验配置的哈希值，确保可复现
- 保存实验环境的快照（requirements.txt、环境变量）

## 总结（总）

实验记录是 Solver 开发与验证的核心环节。通过系统化的实验设计、执行与分析，可以：

- **验证算法正确性**：确保实现符合理论预期
- **优化性能**：找到最佳参数与配置
- **指导开发方向**：基于实验结果决定下一步优化重点

建立完善的实验记录体系，有助于形成可持续的迭代开发流程。
